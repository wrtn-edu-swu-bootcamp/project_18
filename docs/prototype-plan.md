# S2S 재고 관리 챗봇 - 2일 프로토타입 개발 계획

> **목적**: 학교 과제용 실제 동작하는 프로토타입 (2일 내 완성)

## 프로젝트 개요

**형태**: 모바일 반응형 웹 앱 (핸드폰에서도 사용 가능)
**핵심 기능**: 챗봇, 자동 이메일 발송, 재고 상태 관리
**개발 기간**: 2일 (16시간)

## 기술 스택

### 프론트엔드
- **React + Vite**: 빠른 개발 환경
- **Tailwind CSS**: 빠른 스타일링
- **React Router**: 페이지 라우팅

### 백엔드
- **Node.js + Express**: 간단한 API 서버
- **JSON 파일**: 데이터 저장 (DB 불필요)
- **nodemailer**: 실제 이메일 발송

### 배포
- **로컬 실행**: 과제 시연용

## 핵심 기능 (MVP)

### 1. 챗봇 인터페이스

**구현 내용:**
- 카카오톡 스타일 채팅 UI
- 메시지 입력 및 응답 표시
- 키워드 파싱 (정규표현식)
  - 예: "청바지 32인치 5개" → {상품: "청바지 32인치", 수량: 5}

### 2. 재고 매칭 시스템

**구현 내용:**
- 샘플 매장 데이터 (3-5개 매장)
- 키워드 기반 재고 검색
- 보유 매장 카드형 UI
- 매장 선택 기능

### 3. 자동 이메일 발송

**구현 내용:**
- Gmail SMTP 연동
- 재고 요청 메일 자동 발송
- 발송 성공/실패 알림

**이메일 템플릿:**
```
[재고 요청]
요청 매장: 강남점
상품명: 청바지 32인치
수량: 5개
요청 날짜: 2026-01-21
```

### 4. 재고 상태 관리

#### 4.1 현재 재고
- 보유 재고 목록
- 재고 추가/수정 기능

#### 4.2 대기 중 재고 (오는 재고)
- 내가 요청한 재고 목록
- 요청 상태 확인 (요청됨, 승인됨, 완료)
- 요청 매장 정보

#### 4.3 준비 중 재고 (가는 재고)
- 받은 요청 목록
- 승인/거절 버튼
- 상태 업데이트

### 5. 거래 내역
- 전체 거래 히스토리
- 타임라인 형식

## 데이터 구조

### stores.json
```json
{
  "stores": [
    {
      "id": "store-1",
      "name": "강남점",
      "email": "gangnam@store.com",
      "inventory": [
        {
          "id": "item-1",
          "name": "청바지",
          "size": "32인치",
          "quantity": 10
        }
      ]
    }
  ]
}
```

### requests.json
```json
{
  "requests": [
    {
      "id": "req-1",
      "fromStoreId": "store-1",
      "fromStoreName": "강남점",
      "toStoreId": "store-2",
      "toStoreName": "신촌점",
      "item": "청바지 32인치",
      "quantity": 5,
      "status": "requested",
      "type": "outgoing",
      "createdAt": "2026-01-21T10:00:00Z",
      "emailSent": true
    }
  ]
}
```

### 상태 종류
- `requested`: 요청됨
- `approved`: 승인됨
- `in_transit`: 배송 중
- `completed`: 완료
- `rejected`: 거절됨

## 시연 시나리오

### 시나리오 1: 재고 요청 (강남점 → 신촌점)

1. 강남점 직원이 챗봇 접속
2. "청바지 32인치 5개 필요해" 입력
3. 챗봇 파싱: "청바지 32인치를 5개 찾고 계시군요!"
4. 보유 매장 표시: 신촌점 (10개), 홍대점 (7개)
5. 신촌점 선택
6. 실제 이메일 발송
7. 요청 완료 메시지
8. **강남점 앱**: "대기 중 재고"에 추가
9. **신촌점 앱**: "준비 중 재고"에 추가

### 시나리오 2: 요청 승인 (신촌점)

1. 신촌점 직원 앱 접속
2. "준비 중 재고" 메뉴 확인
3. 강남점의 요청 확인
4. "승인" 버튼 클릭
5. 상태 "승인됨"으로 변경
6. 강남점 "대기 중 재고"도 업데이트

### 시나리오 3: 재고 현황 확인

1. **현재 재고**: 보유 재고 목록
2. **대기 중 재고**: 내가 요청한 재고 (예: 신촌점에 청바지 5개)
3. **준비 중 재고**: 받은 요청 (예: 홍대점이 요청한 티셔츠 3개)
