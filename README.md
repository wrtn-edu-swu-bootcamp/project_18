# S2S (Store to Store) 재고 관리 챗봇

> 의류 매장 간 재고 이동을 빠르고 정확하게 처리하는 모바일 챗봇 서비스

## 📋 프로젝트 개요

S2S는 의류 매장 직원들이 매장 간 재고를 요청하고 관리할 때 발생하는 비효율적인 이메일 소통을 해결하기 위한 챗봇 기반 재고 관리 시스템입니다.

### 해결하고자 하는 문제
- ⏰ 메일 작성 및 회신에 시간이 오래 걸림
- ❌ 재고 요청 및 응답이 누락되는 경우가 빈번함
- 📊 재고 현황 파악이 어려움
- 📝 거래 내역 추적 및 관리의 어려움

## ✨ 주요 기능

### 1. 챗봇 기반 재고 요청
- 자연어로 간편하게 재고 요청
- 예시: "청바지 32인치 5개 필요해"
- 보유 매장 자동 검색 및 표시
- 원하는 매장 선택하여 요청

### 2. 자동 이메일 발송
- 재고 요청 시 자동으로 이메일 발송
- 표준화된 템플릿 사용
- 요청 내역 자동 기록

### 3. 재고 상태 관리

#### 현재 재고
- 현재 매장이 보유한 재고 관리
- 재고 등록/수정/조회

#### 대기 중 재고 (오는 재고)
- 내가 다른 매장에 요청한 재고
- 요청 상태 실시간 확인
- 승인/배송/완료 상태 추적

#### 준비 중 재고 (가는 재고)
- 다른 매장이 나에게 요청한 재고
- 승인/거절 기능
- 상태 업데이트

### 4. 거래 내역
- 모든 재고 이동 히스토리 조회
- 요청/응답 구분 표시
- 타임라인 형식

## 📱 사용자 플로우

### 재고 요청 시나리오
1. **강남점** 직원: 챗봇에 "청바지 32인치 5개 필요해" 입력
2. 시스템: 재고 보유 매장 검색 (신촌점 10개, 홍대점 7개)
3. **강남점** 직원: 신촌점 선택
4. 시스템: 자동 이메일 발송 + 재고 상태 업데이트
   - 강남점 앱: "대기 중 재고"에 추가
   - 신촌점 앱: "준비 중 재고"에 추가
5. **신촌점** 직원: "준비 중 재고"에서 요청 확인 후 승인
6. 시스템: 양쪽 매장 상태 "승인됨"으로 업데이트

## 📊 데이터 구조

### 재고 요청 (Request)
```json
{
  "id": "req-1",
  "fromStoreId": "store-1",
  "fromStoreName": "강남점",
  "toStoreId": "store-2",
  "toStoreName": "신촌점",
  "item": "청바지 32인치",
  "quantity": 5,
  "status": "requested",
  "createdAt": "2026-01-21T10:00:00Z",
  "emailSent": true
}
```

### 상태 종류
- `requested`: 요청됨
- `approved`: 승인됨
- `in_transit`: 배송 중
- `completed`: 완료
- `rejected`: 거절됨

## 🚀 빠른 시작

### 설치 및 실행

```bash
# 1. 저장소 클론
git clone https://github.com/wrtn-edu-swu-bootcamp/project_18.git
cd project_18

# 2. 백엔드 설정 및 실행
cd backend
npm install
npm start          # http://localhost:3001

# 3. 새 터미널에서 프론트엔드 실행
cd frontend
npm install
npm run dev        # http://localhost:5173
```

브라우저에서 `http://localhost:5173`을 열어 앱을 사용하세요!

### 이메일 설정 (선택사항)

실제 이메일 발송을 테스트하려면:

1. `backend/.env.example`을 `backend/.env`로 복사
2. Gmail 앱 비밀번호 생성 (https://myaccount.google.com/apppasswords)
3. `.env` 파일에 이메일 정보 입력

자세한 설정 방법은 [SETUP.md](SETUP.md)를 참조하세요.
